## 1. 当前结论

项目已进入“可用 Beta + 后台治理”阶段，重点从“功能打通”转为“稳定性、自动化与可维护性”。

- 下载主链路：可用
- 后台治理：可用（用户管理 + 操作审计 + 登录态管理）
- 通知中心：可用
- 账号安全能力（改密/绑定手机）：可用
- 后续重点：自动化回归、支付闭环、可观测性

## 2. 已完成能力清单

### 2.1 前端

1. 首页链路
- 支持分享文案内 URL 提取
- 解析后统一展示视频信息与下载能力
- 音频选项按“是否存在独立音轨”动态显示

2. 账号体系
- 登录/注册页已重设计（与首页风格统一）
- 用户中心保留：个人资料、下载历史、通知中心、账户设置
- 下载历史支持分页、标题外链跳转、删除与重下

3. 后台管理（仅超级管理员）
- 统一页面：`/admin?tab=users|audit|auth`
- 入口位于顶栏“后台管理”
- `/admin/users` 兼容重定向至 `/admin?tab=users`

4. 用户管理
- 筛选：关键词、角色、会员、账号状态
- 分页默认每页 10 条
- 列表交互优化：文本截断 + hover 完整显示、无横向滚动
- 行内操作：角色调整、会员调整、账号启停（按钮为完整中文语义）
- 默认排序：超级管理员优先，随后按注册时间从早到晚

5. 操作审计
- 记录动作 + 模块 + 平台 + 操作人 + 目标 + 时间
- 支持筛选：模块、平台、关键词、分页
- 展示变更细节（如角色/状态/会员从何值变为何值）

### 2.2 后端

1. 用户模型与权限
- 用户字段已统一：`role`、`legacy-access-field`、`accountStatus`
- 旧的访问等级布尔字段已移除，历史逻辑已迁移到新的等级字段
- 禁用账号不可登录、不可下载

2. 后台接口
- `GET /admin/users`
- `PATCH /admin/users/:id/role`
- `PATCH /admin/users/:id/membership`
- `PATCH /admin/users/:id/status`
- `GET /admin/audit`（主入口）
- `GET /admin/users/audit`（兼容入口）

3. 审计与通知
- 审计字段包含 `module/platform/targetType/beforeState/afterState`
- 用户、角色、登录态关键动作均写入审计
- 站内通知覆盖：
  - Cookie 风险/失效/恢复
  - 会员开通、续费、即将到期、已过期
  - 密码修改、手机号修改
  - 后台用户治理动作

4. 登录态健康检查
- 定时任务周期检查 B站/抖音登录态
- 状态分级：`healthy/degraded/invalid`
- 失败阈值触发失效判定与通知
- 支持手动同步检查 `GET /auth/health?sync=1`

### 2.3 平台能力（真实现状）

1. 抖音：支持解析/预览/下载；具备风控重试、质量探测与降级回退
2. B站：支持解析/预览/下载；高质量依赖登录态
3. 小红书：支持解析/预览/下载；画质档位受源站约束
4. 快手：支持解析/预览/下载；预览优先 H264/HLS 以保证浏览器兼容
5. YouTube：支持解析/预览/下载；高画质异步任务 + `yt-dlp` 失败回退

## 3. 近期关键落地（本轮）

1. 后台管理整合：用户/角色操作合并至用户管理，不再保留独立角色页
2. 用户管理表格体验修复：字段压缩、截断展示、操作可见性优化
3. 筛选与分页修复：普通用户 + VIP 组合筛选结果正确
4. 数据字段治理：彻底移除旧布尔访问等级字段，统一到新等级字段
5. 首页与登录注册视觉重构：统一 V-SAVE 设计语言
6. 品牌与版权统一：项目名、页脚版权、文档命名一致化

## 4. 当前风险与技术债

1. 反爬策略波动风险
- 抖音/快手/小红书源站策略变化会影响稳定性

2. 自动化测试缺口
- 前端缺少关键链路 E2E
- 多平台回归脚本未统一接入日常流水线

3. 支付闭环缺失
- 会员购买流程仍为站内占位实现

4. 可观测性不足
- 下载任务缺少统一指标面板（成功率、耗时、失败码）

## 5. 下一阶段建议（优先级）

1. `P0` 建立多平台回归基线（抖音/小红书/快手/YouTube）
2. `P0` 补齐前端 E2E（解析、预览、下载、后台核心操作）
3. `P1` 建立下载任务可观测性与告警
4. `P1` 接入支付网关与会员回调闭环
5. `P2` 通知通道扩展（站内 + 邮件/短信可配置）
