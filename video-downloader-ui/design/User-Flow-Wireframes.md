# 用户流程与线框（历史设计参考）

**Last Updated:** 2026-03-20  
**Status:** Archived（仅用于回溯设计演进，不作为当前实现依据）

> 当前功能事实以 `README.md` 与 `docs/plans/2026-03-20-development-status.md` 为准。

## 1. 当前生效的核心流程（摘要）

### 1.1 下载主流程（双端）

1. 输入链接或粘贴分享文案
2. 自动提取可识别链接
3. 调用 `/api/download/parse`
4. 展示预览卡片（封面 / 标题 / 作者 / 时长）
5. 选择格式与质量
6. 登录用户发起下载（`/api/download/get-url`，需显式携带 `clientType=WEB|MOBILE`）
7. 成功后记录下载历史

### 1.2 后台管理流程（Web）

入口：顶部导航“后台管理” -> `/admin?tab=runtime`（仅 `SUPER_ADMIN`）

- 默认落到 `运行看板`
- 侧边菜单顺序：运行看板 -> 下载模式管理 -> 登录态管理 -> 用户管理 -> 操作审计
- 登录态管理支持：
  - B站：状态 -> 生成二维码 -> App 扫码 -> 轮询成功 -> 手动刷新 / 清空
  - 抖音：状态 -> 生成二维码或手动 Cookie -> 轮询成功 -> 清空

### 1.3 历史与通知流程（2026-03-20 口径）

- 下载历史：支持平台筛选、日期筛选、一键清空
- iOS 历史：支持长按进入多选、全选、删除选中
- 通知中心：支持全部已读、一键清空
- 登录态类通知“去处理”统一跳转到 `/admin?tab=auth`

### 1.4 后台下载模式管理流程（Web）

入口：`/admin?tab=download-policy`（仅 `SUPER_ADMIN`）

1. 页面按“可编辑平台 / 只读平台”分区展示
2. 支持卡片展开 / 收起与“全部展开 / 全部收起”
3. 在展开态编辑 Web / Mobile 模式并保存，配置实时生效
4. 只读平台仅展示固定策略说明，不提供编辑控件

### 1.5 运行看板链路钻取流程（Web）

1. 进入 `运行看板`
2. 选择时间窗与平台
3. 在“平台全链路接口耗时”中查看最近链路
4. 打开链路详情抽屉
5. 按阶段查看端侧耗时、接口耗时与阶段明细

## 2. 线框说明

## 3. 异常路径（仍有效）

1. 链接无效：输入区直接提示
2. 平台风控：展示可重试文案与登录态引导
3. 视频不可用：展示不可重试文案
4. 未登录下载：跳转登录页
